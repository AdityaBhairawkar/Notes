In Linux, every file or directory has **permissions** that control **who can read, write, or execute** it.

There are **3 types of users**:

|User Type|Meaning|
|---|---|
|`u`|Owner (user who created the file)|
|`g`|Group (users in the file's group)|
|`o`|Others (everyone else)|
|`a`|All (`u+g+o`)|

## ğŸ§¾ Understanding Permission Types

|Symbol|Numeric|Meaning|
|---|---|---|
|`r`|4|Read|
|`w`|2|Write|
|`x`|1|Execute|
|`-`|0|No permission|

Each permission set has 3 parts:

```
rwx rwx rwx
 u   g   o
```

Example: `-rwxr-xr--`

- Owner â†’ `rwx` (read, write, execute)
    
- Group â†’ `r-x` (read, execute)
    
- Others â†’ `r--` (read only)

## ğŸ¯ Changing Permissions

### âœ… Symbolic Mode (using letters)

```bash

chmod g-w filename   # Remove write from group
chmod o=r filename   # Set read-only for others
chmod a+x filename   # Add execute for all
```

âœ… Example:

```bash

```

Means: owner can read/write, group read-only, others no access

---

### âœ… Numeric Mode (using numbers)

```bash

```

How to calculate:

- Owner: 7 = 4+2+1 = `rwx`
    
- Group: 5 = 4+0+1 = `r-x`
    
- Others: 4 = `r--`
    

|Value|Meaning|
|---|---|
|`7`|`rwx`|
|`6`|`rw-`|
|`5`|`r-x`|
|`4`|`r--`|
|`3`|`-wx`|
|`2`|`-w-`|
|`1`|`--x`|
|`0`|`---`|
